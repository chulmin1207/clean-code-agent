현재 변경사항을 클린코드 관점에서 분석하세요.

## 실행 순서

1. `git diff --cached` 로 스테이징된 변경 확인. 없으면 `git diff` 로 언스테이징 변경 확인
2. 변경사항이 없으면 "변경사항 없음 — 분석 대상이 없습니다" 출력 후 종료
3. 변경된 파일만 대상으로 정량 분석:
   - 변경된 함수의 줄 수 측정
   - 새로 추가된 식별자의 명명 검사
   - 변경 범위 내 중첩 깊이 확인
   - 변경된 파일 간 중복 패턴 탐지
4. 문제점을 심각도(CRITICAL/HIGH/MED/LOW) 순 정렬
5. 각 감점에 `파일:라인` 근거 첨부
6. 즉시 수정 가능한 항목은 Before/After 수정안 제시

## 출력

- 변경 없음: "변경사항 없음 — 분석 대상이 없습니다"
- 문제 없음: "✓ 클린코드 기준 통과 ({N}개 파일 검사)"
- 문제 있음:
  ```
  ## /clean 결과: {N}개 파일 검사

  ### 감점 내역
  [관점] -N: 대상 (파일:라인) — 사유
  ...

  ### 문제 목록
  | 심각도 | 파일:라인 | 관점 | 설명 | 수정안 |
  |--------|-----------|------|------|--------|
  ```
