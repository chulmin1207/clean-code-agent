$ARGUMENTS 에 지정된 대상을 클린코드 기준으로 전면 리뷰하세요.

## 입력 판별

- 파일 경로 → 해당 파일 분석
- 디렉토리 경로 → 하위 소스 파일을 Glob으로 수집, 각각 분석 후 종합 리포트
- 인자 없음 → `git diff --name-only HEAD~1` 로 최근 커밋 변경 파일 자동 선택

## 실행 순서

1. 대상 파일 전체 읽기
2. 정량 측정 수행:
   - 각 함수의 줄 수 계산 (시작~끝 라인)
   - 1-2글자 식별자 Grep으로 카운팅 (루프 변수 i/j/k 제외)
   - 각 라인 인덴트 레벨로 중첩 깊이 측정
   - 3줄 이상 동일/유사 패턴 Grep으로 중복 탐지
   - import/require 문 수집하여 의존성 확인
3. 감점 내역을 `[관점] -N: 대상 (위치) — 사유` 형식으로 나열
4. 루브릭에 따라 점수 산출 (근거 없는 감점 금지)
5. 개선 우선순위 상위 3가지 제시

## 디렉토리 분석 시

- 파일별 개별 점수 + 전체 평균 점수 산출
- 프로젝트 전체에 걸친 공통 문제 패턴 식별
- import 그래프 구성 → 순환 의존성 탐지
- 레이어 위반 탐지 (하위→상위 import)
- God 파일 탐지 (300줄 초과 + export 10개 이상)
- 가장 시급한 파일 TOP 3 지목

## 출력에 반드시 포함

1. 감점 내역 (정량 근거 + 파일:라인)
2. 종합 점수
3. 발견된 문제 테이블
4. Before/After 리팩토링 제안
5. 개선 우선순위
